## Rúbrica de Evaluación

*(Todas las actividades pertenecen al mismo proyecto dividido en fases.)*

---

### **Prueba 1 – 10%**

**Instrucción:** Redactar una situación de una entidad que necesita una base de datos (máximo una página) y entregar un archivo **README.md**.

**Matriz de evaluación:**

| Criterio                                    | Satisfactorio                                       | Incompleto                                                | Deficiente                                             |
| ------------------------------------------- | --------------------------------------------------- | --------------------------------------------------------- | ------------------------------------------------------ |
| Claridad y coherencia de la situación       | Presenta una situación clara, coherente y completa. | Presenta la idea general pero con vacíos o poca claridad. | La situación es confusa o no cumple con lo solicitado. |
| Definición de la necesidad de base de datos | Explica claramente la necesidad y el propósito.     | Menciona la necesidad sin desarrollarla.                  | No justifica la necesidad.                             |
| Entrega del README.md                       | README completo y bien estructurado.                | README entregado pero incompleto.                         | No entrega README o está vacío.                        |

---

### **Prueba 2 – 5%**

**Instrucción:** Elaborar un diagrama que represente la base de datos y entregar la imagen del diagrama.

**Matriz de evaluación:**

| Criterio                        | Satisfactorio                                            | Incompleto                                          | Deficiente                          |
| ------------------------------- | -------------------------------------------------------- | --------------------------------------------------- | ----------------------------------- |
| Estructura general del diagrama | Diagramación clara con entidades y relaciones completas. | Diagrama con errores menores o elementos faltantes. | Diagrama incorrecto o no entregado. |
| Relación entre entidades        | Relaciones correctas y coherentes.                       | Relaciones parcialmente correctas.                  | Relaciones incorrectas o ausentes.  |
| Formato de entrega              | Imagen entregada correctamente.                          | Imagen entregada con problemas.                     | No entrega imagen.                  |

---

### **Tarea I – 5%**

**Instrucción:** Crear la estructura de la base de datos y entregar el archivo **.sql** correspondiente.

**Matriz de evaluación:**

| Criterio                                  | Satisfactorio                                               | Incompleto                                | Deficiente                          |
| ----------------------------------------- | ----------------------------------------------------------- | ----------------------------------------- | ----------------------------------- |
| Correcta creación de tablas               | Todas las tablas están completas y correctamente definidas. | Tablas incompletas o con errores menores. | Tablas incorrectas o ausentes.      |
| Integridad de claves primarias y foráneas | Todas definidas correctamente.                              | Algunas definidas incorrectamente.        | No definidas o mal implementadas.   |
| Entrega del archivo SQL                   | Archivo funcional y sin errores.                            | Archivo con errores menores.              | No entrega el archivo o no ejecuta. |

---

### **Tarea II – 5%**

**Instrucción:** Elaborar un documento **README** con las pruebas usadas para validar la base de datos.

**Matriz de evaluación:**

| Criterio                         | Satisfactorio                            | Incompleto                         | Deficiente                             |
| -------------------------------- | ---------------------------------------- | ---------------------------------- | -------------------------------------- |
| Descripción de pruebas           | Pruebas claras, detalladas y coherentes. | Pruebas incompletas o poco claras. | No incluye pruebas o son irrelevantes. |
| Evidencia o ejemplos mencionados | Incluye ejemplos o evidencia pertinente. | Incluye evidencia mínima.          | No aporta evidencia.                   |
| Calidad del README               | Estructurado y entendible.               | Parcialmente organizado.           | Desordenado o no entregado.            |

---

### **Proyecto I – 10%**

**Instrucción:** Realizar las pruebas de calidad de la base de datos.

**Matriz de evaluación:**

| Criterio                        | Satisfactorio                             | Incompleto               | Deficiente                          |
| ------------------------------- | ----------------------------------------- | ------------------------ | ----------------------------------- |
| Pruebas de calidad ejecutadas   | Pruebas completas y bien documentadas.    | Pruebas parciales.       | Pruebas incorrectas o inexistentes. |
| Análisis de resultados          | Conclusiones claras y fundamentadas.      | Análisis superficial.    | No presenta análisis.               |
| Consistencia con etapas previas | Total coherencia con entregas anteriores. | Algunas inconsistencias. | No coincide con lo desarrollado.    |

---

### **Proyecto II – 10%**

**Instrucción:** Elaborar y entregar un video explicando y presentando el trabajo final.

**Matriz de evaluación:**

| Criterio                 | Satisfactorio                                      | Incompleto                             | Deficiente                        |
| ------------------------ | -------------------------------------------------- | -------------------------------------- | --------------------------------- |
| Explicación del proyecto | Explicación clara, completa y coherente.           | Explicación parcial o poco clara.      | Explicación confusa o incompleta. |
| Presentación del proceso | Muestra adecuadamente todas las fases del trabajo. | Muestra solo algunas fases.            | No muestra el proceso.            |
| Calidad del video        | Audio e imagen claros y formato correcto.          | Presenta problemas menores de calidad. | Video defectuoso o no entregado.  |
